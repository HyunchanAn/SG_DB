# SG_DB 데이터베이스 재구축 및 MLOps 파이프라인 통합 기획안 (최종본)

본 문서는 파이프라인 팀(맥북)과 AI 팀(메인 워크스테이션) 간의 논의를 종합하여, 데이터 무손실 적재부터 AI 연쇄 재학습까지 포괄하는 End-to-End 시스템 재구축 마스터 플랜입니다.

---

## 1. 용어 정의 및 도메인 지식 동기화

데이터베이스 스키마 설계의 오류를 방지하기 위해 아래의 핵심 개념을 엄격히 정의합니다.

* **제품 (OurProduct)**: 특정 원단(기재)에 점착제가 도포되어 완성된 **보호 테이프 완제품**(예: SGV218ME)을 의미합니다. 내부에 '점착제'를 속성으로 가집니다.
* **피착재 (Adherend)**: 테이프의 원단이 아니라, 고객사가 보호 테이프를 부착하고자 하는 **대상 표면**(예: SUS304-BA)을 의미합니다. 거칠기, 표면 에너지 등이 점착력 예측의 주요 Feature가 됩니다.
* **점착제 (Adhesive)**:
  * 양산품: 3자리 숫자 (예: 701, 702)
  * 연구실 합성품: `S` + 날짜 + 알파벳 (예: S250421B)

---

## 2. 데이터베이스 재구축 및 스키마 설계 전략

### Step 1: 원시 데이터 중간 정제소(`parsed_data`) 구축 및 파싱
* `SG_DB` 루트 경로에 `parsed_data/` 디렉토리를 신설합니다.
* `raw_data/` 내의 엑셀/CSV 데이터를 파싱 스크립트로 정리하여(예: 두께와 바코터 번호 분리 등) `parsed_data/`에 종류별 CSV로 저장합니다.

### Step 2: 사용자 중간 육안 검증
* 파싱 완료 후, 사용자가 `parsed_data/` 내의 CSV를 직접 열어 누락 데이터 유무 및 파싱 무결성을 검증합니다.

### Step 3: 신규 ERD 설계 및 개발용 DB(`dev_db`) 구축
* 운영 환경과 독립된 개발용 도커 컨테이너(`dev_db`)를 띄웁니다.
* 미래 AI 모델 고도화를 대비하여, 다음과 같은 예비 스키마(Nullable)를 테이블에 반드시 포함합니다.
  * **합성(Synthesis) 이력 테이블**: `분자량(Mw, Mn)`, `분산도(PDI)`, `FT-IR 스펙트럼 매핑 외래키(FK)`.
  * **도포(Coating) 이력 테이블**: `초기 점착력(Ball Tack 등)`, 정량적 `유지력(밀린 거리 등)`, `광학 물성(투과율, Haze)`.

---

## 3. MLOps 및 연쇄 재학습(Cascade Retraining) 파이프라인

### Step 4: 데이터 적재 및 실험 이력 서빙 API 신규 개발
* 정제된 CSV를 `dev_db`에 Load 합니다.
* `SG_proj_004` 모듈을 Repository 패턴으로 리팩토링합니다.
* **AI 학습용 신규 API 개발**: `SG_sys/shared_schemas`를 업데이트하고, `SG_proj_004`에 `/experiments/synthesis`, `/experiments/coating` 엔드포인트를 노출하여 메인 워크스테이션이 데이터를 수집할 수 있게 합니다.

### Step 5: 시스템 정합성 통합 테스트 및 스위칭
* 구형 DB 환경에서 신형 DB(`dev_db`) 환경으로 파이프라인 엔드포인트를 교체합니다.

### Step 6: 1차 AI 모델 재학습 트리거 (`SG_proj_001`)
* DB 스위칭 직후, 맥북 파이프라인에서 트리거를 발생시켜 메인 워크스테이션의 `SG_proj_001` 모듈(XGBoost, CVAE)을 전면 재학습시킵니다. 수년 치의 데이터 분포 변화를 반영하여 Stale Weight를 해소합니다.

### Step 7: 2차 메타 모델 연쇄 재학습 트리거 (`SG_proj_013`)
* `001` 모듈의 재학습 완료(가중치 판단 완료) 신호를 받으면, `001`의 예측값을 입력으로 사용하는 **`SG_proj_013` (역설계 검증 및 앙상블 메타 모델)의 파이프라인을 즉시 연쇄 호출(Cascade Trigger)**합니다.
* 전체 재학습 완료 시, 최종 시스템 E2E 구동 테스트를 진행합니다.
